Proposal: Terminal Game Demos

Author(s): Zach Kohlberg

Program Summary:

  • Several small programs demonstrating various topics, such as File I/O, exception handling, ANSI escape sequences, enums, and networking.
  • One larger program that implements a Wordle-like game, either as a single-player game or, if time permits, as a two-player game played over a network.

Learning Goals:

  • I know relatively little about ANSI escapes and will have fun learning to code some TUI features from scratch.
  • I've written network code in Java before, but I'm curious to see how much I can simplify a networked program to make it accessible to CSC110 students. I think I can avoid some of the topics I'd normally need to cover for such a program:
    • Multithreading isn't necessary if we keep our program to two users and don't try to handle any errors that might occur. This also removes the need for a queue or any concurrent data structures to handle incoming messages or allow users to join/reconnect in the middle of the game.
    • We can avoid introducing any GUI elements if we just print to the terminal, and we can use ANSI escapes to avoid mangling user input with new output.

Concrete Objectives:

  • Separate example programs, each demonstrating a simple usecase for one of the following topics:
    • Exception handling
    • File I/O
    • Enums
    • ANSI escapes
  • A functioning Password/Wordle-like terminal game, probably single-player, that uses some of the above topics (minimally reads a word list file to establish valid word choices)

Additional Goals (time permitting, not part of grading criteria):

  • Example programs covering:
    • Additional reader/writer classes from the standard library
    • Minimal networking code for a two-user chatroom
  • Make the game two-player over a TCP connection

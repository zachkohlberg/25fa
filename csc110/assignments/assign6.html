Assignment 6

In this assignment you will use your functions from the previous assignment to implement part of the dice game. Your program will include the partial rerolling mechanic used in Yahtzee and the ability to score your dice using the rules from Yahtzee's upper categories.

Autograded

This is an autograded assignment. Read the Autograded Assignments page before starting.

This program involves random numbers. Make sure you're following the random number generation requirements for autograded assignments so that your code is compatible with the tests.

Directions

  1. Write this program in a file named DiceGame2.java and start with a copy of your code from the previous assignment.
  2. Remove the "Dice Game" print statement from your main function and divide it into three sections (you can mark the sections with comments): initialization, rerolling, and scoring.
  3. Initialization:
    • Create an int array for your dice and randomize its elements with your rollAllDice function.
    • Create a variable to track your remaining rerolls and initialize it to 2.
  4. Rerolling: create a loop that repeats while the user has at least one reroll left. Inside the loop:
    • Print the dice using printDice and prompt the user for a command.
    • If the user enters "skip", then set their rerolls to 0. This should end the loop.
    • If the user enters "roll", then
      • Ask them which dice to reroll. Assume their answer consists only of letters from 'a' to 'e' and use your parseReroll and rollSomeDice functions to reroll the appropriate dice.
      • Decrease their rerolls by 1.
    • If the user enters anything else, then print an error message.
  5. Scoring:
    • Print the user's dice so they can see the result of their last reroll.
    • Prompt the user for a die value to score (an integer from 1 to 6). If the user enters a number outside this range, then print an error message and repeat the prompt until you get a valid number.
    • Calculate the user's score with calculateScore and print it.

Requirements

The function requirements and the dice format requirements from the previous assignment still apply. New requirements:

  • Your program processes commands in the reroll phase until the user runs out of rerolls. It correctly responds to any number of invalid commands.
  • Your program reads integers in the scoring phase until it reads a number from 1 to 6 and resonds correctly to integers outside this range.
  • Error messages must be formatted as follows:
    • ERROR: The command "<COMMAND>" is not recognized.
    • ERROR: The number "<N>" doesn't appear on a six-sided die.
  • The score at the end of the program must be formatted You scored <SCORE> points!.

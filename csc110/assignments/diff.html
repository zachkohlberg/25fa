Diffing Output

You can use the diff command on your terminal to compare the contents of two text files and see all differences between the files. This is helpful for checking what doesn't match between your program's output and the expected output when looking at the feedback report for an autograded assignment. To do this:

  1. Create two text files. The names don't matter, so for this example we'll say they're called student.txt and expected.txt.
  2. Find the output for a test case you'd like to check in the test report. Copy your program's output to one file (student.txt) and the expected program output to the other file (expected.txt).
  3. Run the command diff followed by the two files (diff student.txt expected.txt).
  4. The output will show any lines on which the files differ.

The output from diff can be difficult to parse, so here's a summary of what the above diff means:

  • A diff shows us a list of changes that would convert one file to the other. Each line starting with a < is a line we would remove from the first file, and each line starting with a > is a line we would add to the first file. In other words, assuming our output was first in the diff command (diff student.txt expected.txt and not diff expected.txt student.txt), < marks incorrect lines from our output and > marks the corresponding correct lines from the expected output.
  • Line numbers are listed before the < and > lines as "XcY". X is a line number from our file and Y is a line number in the expected output file. X and Y can also be ranges if that part of the diff deals with multiple lines.
  • For example, the first pair of lines (marked 5c5, indicating that these are line 5 in our output and the expected output) tells us that our output is missing the quotes around roooooolllll. The incorrect line from our output doesn't have the quotes, and the correct line from the expected output does have the quotes. The sections labeled 10c10 and 15c15 tell us the same thing.
  • Further down, the section marked 22c22,26 is telling us that line 22 from out output file is incorrect and the correct output includes lines 22 to 26 from the expected output file. There are two problems here: our output ends in a 5 rather than a 2, and it's missing a second printing of the dice. The missing output is why diff wants to replace our single line with five lines from the expected output. One line to fix the mistake and four more lines for the extra printing of the dice we were missing.
  • Finally, the section marked 24c28 shows us that our final line of output (line 24 in our output file and line 28 in the expected output file) is missing an exclamation point.

The diff command is most useful when you have a small number of minor differences between your output and the expected output. In that case, the diff will be short and easy to read. If the outputs are nothing alike, then the diff will just show all or most of each file, which doesn't tell you anything you didn't already know.

Autograded Assignments

Testing

Autograded assignments are subject to the submission limits given in the deadline policy and will be checked by several programs I wrote to test student code. These tests will run your program or its functions with different inputs and check whether it produces the correct results.

Academic Integrity and Accountability

Attempting to deliberately fool the test programs, hide malicious code in your source file, or otherwise intentionally sabotage tests is not allowed and will be treated as an AIA violation.

Test Criteria

Tests will normally check the following:

  • Output produced by IO.println and IO.print. Your program's output must match any requirements given in the assignment. Any difference (wrong spelling, capitalization, etc.) will fail an I/O test.
  • Values returned by your function. If your function returns a value, then the value must match any requirements given in the assignment.
  • Modifications to array arguments passed to your function. If your function takes any array parameters, then it must modify them as described in the assignment's requirements. Any extra modifications will usually cause a test to fail.

Tests will always ignore any output produced by IO.readln. I never specify how exactly to prompt the user for input, so you may word these prompts however you want. The tests will only look at output from IO.print and IO.println. This also means you shouldn't use IO.readln to print any of the required output.

A few other ways for a test to fail:

  • Prompting for additional input or prompting in the wrong order. The test provides input in the order described in the assignment, so your code needs to read it in the same order.
  • Printing output in the wrong order, combining separate lines into the same line, or splitting up one line into several lines.

Random Numbers

If your program needs to generate random numbers, then it must use Math.random() to generate the number and scale, adjust, and round this value as needed. For example, (int)(Math.random() * 6) + 1 generates a random integer from 1 to 6.

Your program must also avoid generating unnecessary random numbers. The tests will seed the random number generator to ensure your code receives the same sequence of "random" values that were used when generating the expected output, and generating extra random numbers in your code will change this sequence and cause you to fail tests.

Test Feedback

My test program will generate markdown, html, and pdf files with the results of any failed tests. This feedback will provide some information about the failed test to help you find and fix any mistakes in your program.

Code Restrictions

To ensure your code is compatible with my tests, there are a few restrictions you need to follow. Ignoring any of these restrictions will usually cause your submission to fail all tests (or be marked invalid before even reaching the tests), and in some cases it will flag your code for review as potentially plagiarized or AI-generated.

These restrictions aren't a problem if you are writing your code as shown in class and only write code that you understand. If you don't know what something means and we haven't covered it in class (like a static modifier), then there's no reason you'd write it into your program.

Compact Source

You program must be a compact source file, and your main function must be declared as void main(). You must not use static or any other modifiers with your functions and variables.

I/O Functions

Any I/O must use the IO functions we learned in class: IO.readln, IO.print, and IO.println. No System.out, Console, or Scanner.

Standard Library

The Java standard library is a useful tool, and you should learn how to use it. However, these projects are meant to be completed with the limited tools we cover in class, and the standard library can trivialize some problems that you're meant to solve by hand. You're free to make use of any String methods, any of the IO and Math functions we've covered, and parsing functions (like Integer.parseInt). However, using other parts of the standard library that we haven't covered, such as System or Scanner, is not allowed. You're also not allowed to use any third-party libraries in autograded assignments.

Java Keywords/Features

Your code should not declare classes, use try-catch, or otherwise rely on advanced features of the language that we haven't covered in class. The tools you have are more than adequate to solve these problems, and they allow for a variety of solutions to most of the programs.

Character Sheet Project

This is some background information for the next four assignments. If you're not interested in the background and just want to write the programs, then feel free to skip this! You can code those programs without knowing anything about TTRPGs.

Dungeons and Dragons

Dungeons and Dragons is a tabletop roleplaying game (TTRPG). Even if you haven't played a TTRPG before, you've probably still heard the name "Dungeons and Dragons" at some point. These assignments will use a modified version of some character creation rules from D&D 5e. The real rules can be found in the 5e System Reference Document (SRD), which you can read if you're curious and want to learn more. However, you do not need to read the SRD for this assignment.

Character Sheet

The idea behind these assignments is to write a program that generates a "character sheet" for someone playing a TTRPG like D&D. A player normally controls a fictional character (their Player Character, or PC), and they track information about their character on a character sheet. This might include statistics, like how strong or fast the character is, the character's possessions, or information about the character's personality.

The character sheet you generate will not cover all of the information about a real TTRPG character. This would get too complicated for an early programming assignment. Instead, we'll use a small subset of rules from the SRD, some of which will be simplified to better fit this assignment. You don't need to know any of these rules! The assignments will explain any calculations your code needs to perform.

Terminology

Below is a summary of some terms that will come up in this assignment:

  • A character's class is the role they fill in the game's world. In a game like D&D, which focuses on fighting fantasy monsters, a character's class tells us their role in combat. A wizard casts spells, a rogue sneaks up on enemies, and a fighter... fights. Yes, everyone fights, but the fighter fights more directly, like with a sword and shield.
  • A character's ancestry or race (the term varies by game, but I'll usually default to ancestry) represents what kind of creature they are. D&D includes a lot of your standard fantasy races: elf, human, dwarf, goblin, gnome, halfling, and orc. However, depending on the game and who's running it, a TTRPG character could be anything: a ghost, a dragon, a talking bear, or a mannequin. I have run a game of D&D that included both a talking bear and a mannequin as player characters. The mannequin also talked, and his name was Bob. The bear had a pet monkey. All of this will be on the exam1.
  • A character's hit points (HP) represent how difficult they are to defeat in combat.
  • A character's armor class (AC) represents how hard they are to hit with an attack. A high AC may mean that a character is very good at avoiding attacks, or that a character is wearing armor that blocks most attacks.
  • A character's ability scores are numbers that represent a character's strength, cunning, speed, and other innate qualities. These affect most actions in the game.
  • A character's ability modifiers are bonuses or penalties derived from their ability scores. The modifiers are the numbers we actually use in most calculations, and the scores only exist to determine the modifiers. At this point, you may be wondering why we bother with ability scores if we don't use them after calculating the modifiers. Good question! Anyway, let's talk about the character's level.
  • A character's level represents how powerful and experienced they are. In most TTRPGs, characters gain experience and level up to represent how they've grown during their adventures.
  • In addition to standard six-sided dice (referred to as D6 dice), D&D uses several less common dice shapes: D4, D8, D10, D12, and D20.

Assignment 1

In this assignment you will write a program that prompts the user for information and prints this information to the terminal. You'll also perform a few operations to show that the information was parsed as the correct type.

Autograded

This is an autograded assignment. Read the Autograded Assignments page before starting.

Directions

  1. Write this program in a file named CharacterSheet1.java.
  2. Prompt the user for the following information (in order):
    • The player's name (String)
    • The character's name (String)
    • The character's class (String)
    • The character's ancestry (String)
    • The character's level (int)
    • The character's strength score (int)
    • The character's dexterity score (int)
    • The character's constitution score (int)
  3. Print each of these values back in the same order (player name, character name, etc.). Each value should be printed on a separate line. Print only the value, no label!
  4. Print the sum of the level, strength score, dexterity score, and constitution score.

One problem you might run into with this program is that "class" is a Java keyword. It would be natural to use class as the name for the variable that holds the character's class, but Java won't allow keywords as identifiers. The name "clazz" is sometimes used as a workaround, but any other name that still conveys the purpose of the variable is fine.

Assignment 2

In this assignment you will add some formatting to your output and perform some calculations that would be used in a real 5e character sheet according to the SRD (see background page for more information).

Autograded

This is an autograded assignment. Read the Autograded Assignments page before starting.

Directions

These directions will use arrow brackets (< and >) to mark placeholder values in the output. Your code should print the relevant value instead of the arrow brackets and whatever text is between them.

  1. Write this program in a file named CharacterSheet2.java.
  2. Prompt for the same input as the previous assignment.
  3. Calculate the character's proficiency bonus using the formula in the requirements section.
  4. For each ability score (strength, dexterity, constitution), calculate a corresponding ability modifier using the formula in the requirements section.
  5. Print the player's name formatted as Player: <NAME>. For example, if the player's name is "Bob" then your code would print "Player: Bob".
  6. Print the character's name, level, ancestry, and class formatted as <NAME>, Level <LEVEL> <ANCESTRY> <CLASS>. For example, if the character's name was "Elrond", their level was 20, their ancestry was "Elf", and their class was "Cleric", then your code would print "Elrond, Level 20 Elf Cleric".
  7. Print the proficiency bonus in the format Proficiency: <PROFICIENCY>. For example, if the character's level is 7, then their proficiency bonus is 3 (using the aforementioned formula) and your code will print "Proficiency: 3".
  8. Print each ability name ("Strength", "Dexterity", "Constitution"), score, and modifier in the following format: <NAME>: <SCORE>, <MODIFIER>. For example, if the character's strength score is 15, then their modifier is 2 (using the aforementioned formula) and your code will print "Strength: 15, 2". Each ability must be printed on a separate line.

Submission

Upload your CharacterSheet2.java file using the submission link on Blackboard.

Requirements

Assignment 3

In this assignment you will implement a point buy system for selecting the user's ability scores and calculate some more stats for the character. The point buy rules will be similar to but simpler than the rules given in the SRD.

Autograded

This is an autograded assignment. Read the Autograded Assignments page before starting.

Directions

The requirements section will show the exact wording required for any warning/error messages printed by your code and the formulas required for any calculations. Anything not specified in this assignment should remain as it was in the previous assignment.

  1. Write this program in a file named CharacterSheet3.java.

  2. Copy your work from the previous assignment into your file. This assignment assumes you are starting from a working version of that program that passes all of its tests.

  3. After the user inputs their class, check whether they chose one of the following options: "Fighter", "Rogue", or "Barbarian". If they chose a different class, then print a warning message to let them know their class isn't supported by our program.

  4. After the user inputs their ancestry, check whether they chose one of the following options: "Orc", "Elf", or "Gnome". If they chose a different ancestry, then print a warning message to let them know their ancestry isn't supported by our program.

  5. After you prompt the user for their ability scores, check whether those scores sum to more than 36. If they sum to more than 36, then print an error message and prompt the user for new ability scores. Repeat this process until the user enters a set of scores that sum to 36 or less.

  6. After the user enters a set of valid ability scores and before you calculate ability modifiers, make the following adjustment based on the character's ancestry:

    • Orc: increase their strength score by 2
    • Elf: increase their dexterity score by 2
    • Gnome: increase their constitution score by 2
    • Other (they chose an unsupported ancestry): don't change any scores

    Note: it's okay if this raises the total above 36! We're only enforcing that rule for the unadjusted scores we get from the user in the previous step.

  7. Determine the "hit die" depending on the character's class:

    • Fighter: 10
    • Rogue: 8
    • Barbarian: 12
    • Other (they chose an unsupported class): 6

  8. Calculate the character's proficiency bonus, ability modifiers, hit points, armor class, and attack using the formulas given in the requirements. Those last three depend on the ability modifiers and proficiency, so calculate them last.

  9. When printing the character sheet, add the following three lines before the proficiency:

    • Hit Points(D<HIT DIE>): <HIT POINTS>
    • Armor Class: <ARMOR CLASS>
    • Attack: <ATTACK>

Submission

Upload your CharacterSheet3.java file using the submission link on Blackboard.

Requirements

  • Prompt for input in the same order as the previous assignment, but with the added loop to enforce the maxmimum ability score total described in the directions.
  • Print the following warning/error messages where appropriate, using the erroneous input instead of the placeholder value. The quotes must still appear in your output.
    • WARNING: The class "<CLASS>" is not supported by this program.
    • WARNING: The ancestry "<ANCESTRY>" is not supported by this program.

    • proficiency bonus: (level + 3) / 4 + 1
    • ability modifier: score / 2 - 5
    • hit points: level * (hit die + constitution modifier)

    • attack: proficiency bonus + strength modifier
  • You may assume the user always enters their class and ancestry capitalized as shown in the directions. Your program doesn't need to be case-insensitive as long as it recognizes "Fighter", "Rogue", "Barbarian", "Orc", "Elf", and "Gnome".

Assignment 4

In this assignment you will allow the user to choose whether to use the previous point buy system or generate a set of random scores by simulating dice rolls. You will also break these parts of your program into separate functions to make them easier to test.

Autograded

This is an autograded assignment. Read the Autograded Assignments page before starting.

This program involves random numbers. Make sure you're following the random number generation requirements for autograded assignments so that your code is compatible with the tests.

Directions

  1. Write this program in a file named CharacterSheet4.java.
  2. Copy your work from the previous assignment into your file. This assignment assumes you are starting from a working version of that program that passes all of its tests.
  3. Write a function named buyScores that takes no parameters and returns an array of ints. This function must perform the same process used in the old program for point buy, add the final scores to an array, and return the array. You can cut and paste that code from your old main function into this new buyScores function, then create and return the array after the loop.
  4. Write a function named roll that takes no parameters and returns an int. This function must generate four random ints from 1 to 6, then return the sum of the three largest numbers it generated. For example, if you generated the numbers 4, 6, 2, and 5 then you would return 15 (4 + 6 + 5), ignoring the 2. If there is a tie for lowest, then just ignore one of those values: 1, 1, 5, and 4 would result in 10 (1 + 5 + 4).
  5. Write a function named rollScores that takes no parameters and returns an array of ints. It must call the roll function three times, put the results into an array, and return the array.
  6. In your main function, remove the old ability score prompts and ask the user whether they want to choose their scores. If they enter y, then generate their scores by calling the buyScores function. If they enter n, then generate their scores by calling the rollScores function. If they enter anything else, then print an error message and ask again until they enter y or n. Remember to store the return value from buyScores or rollScores in a variable and copy the elements of that array to your ability score variables. If you don't do this, then the rest of your program will no longer work!

Upload your CharacterSheet4.java file using the submission link on Blackboard.

Requirements

  • Prompt for input in the same order as the previous assignment, but with the added loop to choose whether to use point buy. If the user doesn't choose point buy, then your program must not prompt the user for their ability scores.
  • Print the following warning/error messages where appropriate, using the erroneous input instead of the placeholder value. The quotes must still appear in your output.
    • WARNING: The class "<CLASS>" is not supported by this program.
    • WARNING: The ancestry "<ANCESTRY>" is not supported by this program.
    • ERROR: You need to answer y or n.

Functions

Your program must implement the following functions and should call them where appropriate.

  • buyScores
    • parameters: none
    • return type: array of int
    • This function must prompt the user for their ability scores until they enter a set of scores that add up to 36 or less. It must also print the appropriate error message each time they enter scores that add up to more than 36. This process is exactly the same as in the previous assignment.
    • The returned array must contain the last set of scores entered by the user, which means its length will always be 3.
  • rollScores
    • parameters: none
    • return type: array of int
    • This function must generate three random scores by calling the roll function.
    • The returned array must contain the three scores generated by calling roll and have a length of 3.
  • roll
    • parameters: none
    • return type: int
    • This function must generate four random ints from 1 to 6 and return the sum of the three highest of those numbers.

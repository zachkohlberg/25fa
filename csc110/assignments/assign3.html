CSC110 Assignments

Assignment 3

In this assignment you will implement a point buy system for selecting the user's ability scores and calculate some more stats for the character. The point buy rules will be similar to but simpler than the rules given in the SRD.

Autograded

This is an autograded assignment. Read the Autograded Assignments page before starting.

Directions

The requirements section will show the exact wording required for any warning/error messages printed by your code and the formulas required for any calculations. Anything not specified in this assignment should remain as it was in the previous assignment.

  1. Write this program in a file named CharacterSheet3.java.

  2. Copy your work from the previous assignment into your file. This assignment assumes you are starting from a working version of that program that passes all of its tests.

  3. After the user inputs their class, check whether they chose one of the following options: "Fighter", "Rogue", or "Barbarian". If they chose a different class, then print a warning message to let them know their class isn't supported by our program.

  4. After the user inputs their ancestry, check whether they chose one of the following options: "Orc", "Elf", or "Gnome". If they chose a different ancestry, then print a warning message to let them know their ancestry isn't supported by our program.

  5. After you prompt the user for their ability scores, check whether those scores sum to more than 36. If they sum to more than 36, then print an error message and prompt the user for new ability scores. Repeat this process until the user enters a set of scores that sum to 36 or less.

  6. After the user enters a set of valid ability scores and before you calculate ability modifiers, make the following adjustment based on the character's ancestry:

    • Orc: increase their strength score by 2
    • Elf: increase their dexterity score by 2
    • Gnome: increase their constitution score by 2
    • Other (they chose an unsupported ancestry): don't change any scores

    Note: it's okay if this raises the total above 36! We're only enforcing that rule for the unadjusted scores we get from the user in the previous step.

  7. Determine the "hit die" depending on the character's class:

    • Fighter: 10
    • Rogue: 8
    • Barbarian: 12
    • Other (they chose an unsupported class): 6

  8. Calculate the character's proficiency bonus, ability modifiers, hit points, armor class, and attack using the formulas given in the requirements. Those last three depend on the ability modifiers and proficiency, so calculate them last.

  9. When printing the character sheet, add the following three lines before the proficiency:

    • Hit Points(D<HIT DIE>): <HIT POINTS>
    • Armor Class: <ARMOR CLASS>
    • Attack: <ATTACK>

Requirements

  • Prompt for input in the same order as the previous assignment, but with the added loop to enforce the maxmimum ability score total described in the directions.
  • Print the following warning/error messages where appropriate, using the erroneous input instead of the placeholder value. The quotes must still appear in your output.
    • WARNING: The class "<CLASS>" is not supported by this program.
    • WARNING: The ancestry "<ANCESTRY>" is not supported by this program.
    • ERROR: Ability scores must sum to 36 or less! Try again.
  • Use the following formulas for your calculations:
    • proficiency bonus: (level + 3) / 4 + 1
    • ability modifier: score / 2 - 5
    • hit points: level * (hit die + constitution modifier)
    • armor class: 10 + proficiency bonus + dexterity modifier
    • attack: proficiency bonus + strength modifier
  • You may assume the user always enters their class and ancestry capitalized as shown in the directions. Your program doesn't need to be case-insensitive as long as it recognizes "Fighter", "Rogue", "Barbarian", "Orc", "Elf", and "Gnome".

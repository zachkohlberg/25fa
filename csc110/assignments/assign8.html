CSC110 Assignments

Assignment 8

This assignment will add multiple rounds and persistent scoring between rounds. You can finally play a full game (of a partial Yahtzee implementation) and keep track of scores from previous rounds.

Autograded

This is an autograded assignment. Read the Autograded Assignments page before starting.

This program involves random numbers. Make sure you're following the random number generation requirements for autograded assignments so that your code is compatible with the tests.

Directions

  1. Write this program in a file named DiceGame4.java and start with a copy of your code from the previous assignment.
  2. Modify your printScores function to take two parameters: an array of ten scores and an array of five dice. The array of scores represents scores from previous rounds, with a negative value indicating a category that hasn't been selected. When you print each category:
    • if the score in the array isn't negative, show that number instead of calculating a score with the dice
    • if the score in the array is negative, show "available (X)", where X is the score calculated for the given dice, to indicate that the user can choose this category and would receive the amount of points shown
  3. Modify your playRound function to take a parameter for the scores array. You'll need to pass this array when you print the scores. Make the following changes to the scoring part of the function:
    • After printing the dice and before prompting for a score category, call your printScores function to show which categories are available and how many points they're worth.
    • Change your input validation to ensure the user chooses a category from 1 to 10 with a negative score. Remember to subtract 1 from the category number before using it to index the scores array (categories are 1-10, but score indexes are 0-9).
    • Change the error message to ERROR: You cannot score category X., where X is the user's invalid input. The old message doesn't make sense now that we're choosing from ten categories instead of six die values.
    • Store the user's score in the scores array at the appropriate index.
  4. Modify your main function to do the following:
    • Create an array of ten scores and initialize each element to -1.
    • Call your playRound function ten times (use a loop), passing it the scores array as an argument. It will add one score to the array with each round, replacing the negative number.
    • Find the total score after ten rounds and print Game Over! Your total score is: X., replacing X with the total score.
    • Print the score categories one more time. You'll need to pass it an array for the dice, even though it won't use the array, so just make a new temporary array and pass that as the second argument.

Requirements

Requirements are the same as previous assignments except as noted here and in the directions.

I/O

Scores

The printScores function must show which categories are available (negative value in the scores array) along with a score calculated for the given dice. For other categories, it must show the score in the scores array. The sample below is for a score array of -1, -1, -1, 12, 5, -1, 23, -1, 50, 11 and a dice array of 6, 6, 6, 5, 6.

Score Categories:

1. Aces: available (0)
2. Twos: available (0)
3. Threes: available (0)
4. Fours: 12
5. Fives: 5
6. Sixes: available (24)
7. 3 of a Kind: 23
8. 4 of a Kind: available (29)
9. Yahtzee: 50
10. Chance: 11

main

Most of your output will be produced by playRound and the functions it calls. Your main only needs to produce two pieces of output at the end of the game. First, the message showing your final score:

Game Over! Your total score is: 166.

Then the printScores output to show the final score in each category:

Score Categories:

1. Aces: 2
2. Twos: 8
3. Threes: 6
4. Fours: 12
5. Fives: 5
6. Sixes: 24
7. 3 of a Kind: 23
8. 4 of a Kind: 25
9. Yahtzee: 50
10. Chance: 11

playRound

Your playRound function needs to produce output in the following order:

  • In each iteration of the reroll phase (this is exactly the same as in previous versions, but I'm including it here for completeness):
    • Print the dice
    • If the user entered an invalid command, print the appropriate error message
  • At the beginning of the score phase (before prompting the user for any input):
    • Print the dice
    • Print the scores
  • For each invalid input received while prompting for a score category:
    • Print the appropriate error message
  • After getting a valid score category:
    • The message announcing how many points the user received

Functions

  • printScores
    • parameters: an int array representing ten scores and an int array representing five dice
    • return type: void
    • This function must print the score categories as specified in the I/O requirements.
  • playRound
    • parameters: an int array representing ten scores
    • return type: void
    • This function must play a round of the game as described in the directions, updating the score array with a new score.
  • main
    • parameters: none
    • return type: void
    • This function must play all ten rounds and print the scores at the end as described in the directions.

Variables

This program prints the standard path to a home directory on various operating systems if "zach" is your username.

// Home.java
void main() {
    IO.println("  Linux: /home/zach");
    IO.println("    Mac: /Users/zach");
    IO.println("Windows: /c/Users/zach");
    IO.println("      or C:\\Users\\zach");
}
// We'll talk about why "\\" prints as "\" later

The Program, Computer, and User

  • Challenging to think about programs:
    • What goes in your code?
    • What does the computer do for you?
    • What information does the user provide?
  • Common mistake: writing user input in your code
  • Three roles to consider:
    • Programmers write instructions
    • Computers follow instructions
    • Users break things provide input
  • To write good instructions, you need to think about how the computer will follow them and how the user will interact with your program. However, none of that actually goes in your code!

Mental Model: I/O and Variables

  • Computers are complicated. How do we understand what they're doing?
  • What can we do that's similar to the following:
    • IO.println
    • IO.readln
    • var x = ...
    • x = ...
  • Easier to understand programs if you can imagine how you would execute their instructions.

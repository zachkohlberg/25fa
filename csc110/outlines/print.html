The Program, Computer, and User

  • Challenging to think about programs:
    • What goes in your code?
    • What does the computer do for you?
    • What information does the user provide?
  • Common mistake: writing user input in your code
  • Three roles to consider:
    • Programmers write instructions
    • Computers follow instructions
    • Users break things provide input
  • To write good instructions, you need to think about how the computer will follow them and how the user will interact with your program. However, none of that actually goes in your code!

Mental Model: I/O and Variables

  • Computers are complicated. How do we understand what they're doing?
  • What can we do that's similar to the following:
    • IO.println
    • IO.readln
    • var x = ...
    • x = ...
  • Easier to understand programs if you can imagine how you would execute their instructions.

Example Programs

// Arithmetic.java
void main() {
    // Input
    int x = Integer.parseInt(IO.readln("Please enter a value for x: "));
    int y = Integer.parseInt(IO.readln("Please enter a value for y: "));

    // Calculations
    // Declaring variables without initializing them
    int sum, difference, product, quotient, remainder;
    // Initializing declared variables
    sum = x + y;
    difference = x - y;
    product = x * y;
    quotient = x / y;
    remainder = x % y;

    // Output
    IO.println("x + y = " + sum);
    IO.println("x - y = " + difference);
    IO.println("x * y = " + product);
    IO.println("x / y = " + quotient);
    IO.println("x % y = " + remainder);
}

// Questions:
// - What happens if we try to print those variables without initializing them?
// - What happens if we write "int sum = x + y" instead of "sum = x + y"? When
//   is it correct to state a variable's type?
// - What happens when the second number is zero?
// - What happens if the user enters a word instead of a number?
// - What happens if you enter a non-integer number, like 2.5?
// - Try changing the program to use doubles instead of integers. How does that
//   change the answers to the last three questions? Remember to use
//   Double.parseDouble instead of Integer.parseInt.

// Height.java
void main() {
    // Input
    int totalInches = Integer.parseInt(IO.readln("How tall are you in inches?"));

    // Calculations
    int feet = totalInches / 12;
    int inches = totalInches % 12;

    // Output
    IO.println("Your height is " + feet + "'" + inches + "\".");
}

Practice

  1. Experiment with the Arithmetic.java program and answer the questions in the comments.
  2. Try writing a program that generates a shorter school email address: fmlast@quinnipiac.edu (first initial, middle initial, last name) instead of first.last@quinnipiac.edu. Ask for the user's first name, middle initial, last name, and school name. Use toLowerCase to ensure nothing is capitalized, and use substring to get the user's first initial.
  3. Try writing a program that calculates the total minutes from a number of days, hours, and minutes. For example, if a user inputs 4 days, 17 hours, and 11 minutes, then the program should respond with "6791 total minutes."
  4. Try writing a program that does the opposite of the previous program: take a number of minutes and convert it to days, hours, and minutes. For example, if a user inputs 6791 minutes, then the program should respond with "4 days, 17 hours, and 11 minutes." This process is similar to the Height.java program.

CSC110 Lecture Outlines

Practice

  1. Answer the questions in the Functions.java example.
  2. Take an earlier program you wrote that follows the pattern "read input, calculate result, print result" and answer the following questions:
    • For any input your program requires, what is that input's type and what name did you use for its variable?
    • If the result of your program had to be stored in a single variable, what would be the most appropriate type for that variable?
    • Rewrite your program as a function that doesn't call any IO functions. Instead, take the input by declaring parameters and return the result with a return statement.
    • Write a main function that prompts the user for input, calls your new function, and prints the result of that function. Your main function is allowed to call IO functions.
  3. Write a clamp function that takes three integers: a value, a minimum, and a maximum. Return the value if it is between the min and max, return the minimum if the value is less than the minimum, and return the maximum if the value is greater than the maximum. You may assume that the minimum is greater than the maximum (i.e. don't worry if someone calls your function with a min of 5 and a max of 0). Do not call any IO or Math functions! There are Math functions that can do all or part of this for you (Math.min, Math.max, Math.clamp), but I want you to write that logic yourself.
  4. Write a program that prompts the user for two numbers, multiplies them, and clamps the result between 0 and 100. Your program should call the clamp function you wrote in the previous problem (copy the clamp function into your program).
  5. Write a program that prompts a user for five numbers and store them in an array. Clamp each number in the array to be between -50 and 50 (modify the contents of the array), then print the array (see Arrays.java for an example of printing an array).

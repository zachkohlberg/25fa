Practice

Number Bases

Rather than giving you a set of problems, I want to suggest a few types of practice exercises that are easy to generate and offer some advice. Your goal in practicing number bases should be to make yourself comfortable with the topics we covered in the last few lectures and develop an intuition for how binary and hexadecimal numbers work. Whatever (reasonable) amount of practice gets you there is what you should do.

Some practice exercises that are easy to generate. You can choose arbitrary digits and generate "random" numbers in whatever base you're using to practice. You can check the decimal value of binary and hexadecimal numbers in jshell by entering the number prefixed with a 0b (binary) or 0x (hex). Binary numbers with more than 32 digits and hexadecimal numbers with more than 8 digits will also need a L added to the end because they're too large for the int type to represent.

  1. Addition: generate two binary or hexadecimal numbers and try adding them by hand using the same addition algorithm you would use in base-10.
  2. Conversions: generate a binary, decimal, or hexadecimal number and try converting it to the other two bases using whichever method you prefer.
  3. Two's complement: generate an 8-bit binary number and find its two's complement. You can check these using the integer.exposed website we used earlier: the signed values should be opposites (except for 0 and -128, which are their own 8-bit two's complement).

Programming

Some programming exercises to try:

  1. Write a program that prompts the user for a number and uses the division method to construct the binary representation of that number. Your program should show each division performed during this process and the result of that division (quotient and remainder), then the actual answer.
  2. Write a program that prompts the user for a binary number (read it as a String, don't try parsing it) and prints the base expansion of that number along with its value in decimal. For example, if the user entered 10110, then your program should print something like 1x2^4 + 0x2^3 + 1x2^2 + 1x2^1 + 0x2^0 = 22. You could also try printing multiple steps of simplification, like 2^4 + 2^2 + 2^1 and 16 + 4 + 2 before showing the decimal value.
  3. (optional) Try writing hexadecimal versions of the previous two programs.

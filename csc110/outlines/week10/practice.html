Practice

With a group or a partner, try designing a program that reads the following information:

  • A list of student names
  • A list of assignment weights that add up to 1.0
  • For each student, a list of grades from 0.0 to 100.0 on every assignment

You can read each list as one line of input and use the .split method covered last week to break it into an array.

After reading and parsing all of the input, print the average grade on each assignment and the weighted average for each student.

Try designing this program like we designed the rock, paper, scissors program: write the main function and make up a function call for each task you need to perform. Make sure you clearly define the name, parameter types, and return type when you write these function calls. Split the functions between yourselves and write the functions separately before combining them into the full program.

After you finish writing the program, discuss the following questions with your group:

  1. What was challenging about writing the main function?
  2. What was challenging about writing your separate functions?
  3. What was challenging about combining the functions into a working program?
  4. Which part of this process was the easiest? Which was the hardest? Why?
  5. If you were to do this again (same process, but a new program), what do you think you could do differently to make it easier?

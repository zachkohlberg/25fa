I/O Over a Network

  • we'll cover a very limited example, not everything required to produce a robust client-server application
  • server: the host; accepts connections from clients
  • client: connects to a server
  • our program:
    • a server that accepts a single client connection
    • once connection is created, both programs behave the same (no difference between client and server)
    • can send and receive chat messages over network
  • we need to use two classes: ServerSocket and Socket
    • a ServerSocket accepts connections from clients and creates Sockets to communicate with those clients
    • a Socket contains an InputStream and an OutputStream for communication with another process over a network
  • some steps to make a more robust/complex program:
    • make a gui or use ANSI to prevent new messages from printing over user's input
    • allow multiple clients to connect:
      • store sockets in a data structure (list or queue)
      • iterate over sockets to read messages from all clients
      • check server socket for new connections
      • remove sockets for disconnected clients
    • handle exceptions to avoid crashing on errors
    • multiple threads:
      • dedicated thread for the server socket to receive new connections
      • separate threads to listen for messages from clients
      • thread safety: use concurrent data structures where appropriate
    • server is the control center
      • clients send input/commands to server
      • server decides how to handle client input
      • server sends results/output to clients

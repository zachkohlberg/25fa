Readers and Writers

  • java standard library has many classes for handling input and output, such as:
    • Scanner
    • PrintStream
    • InputStream
    • BufferedReader
    • BufferedWriter
    • FileWriter
    • PrintWriter
  • these provide different features and deal with i/o at different levels of abstraction
    • InputStream is a lower-level API: it works with raw bytes and isn't ideal if you want strings
    • PrintStream and Scanner are higher-level APIs: they can deal with strings and handle the conversion to bytes and the use of buffers
  • some of these classes throw checked exceptions in their constructors and/or methods, which we need to throw or handle
  • many of these classes mention buffers
    • reading/writing is slow and requires interacting with the operating system (system calls)
    • a buffer is a chunk of memory (such as an array) that can be used to collect data from multiple operations
    • printing can write data to a buffer until it is full, then the whole buffer is written to the real destination (such as a file or standard output) at once to reduce the number of expensive system calls
    • reading can pull a large amount of data into a buffer (one system call) and read a portion of the buffer every time your code calls a read function, reducing the number of times we need to pull new data into the program
    • most of Java's i/o classes will handle buffering for you

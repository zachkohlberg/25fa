Software Installation

This class requires you to have a terminal, text editor, and the Java 25 JDK (or Java 24 until Java 25 is officially released).

Terminal

If you're on Windows, install Git for Windows, which includes Git Bash. The terminal programs that come with Windows are not compatible with some of the terminal commands you'll see in class. You'll need a terminal that runs bash or a similar shell in order to follow along with my examples.

If you're on Linux or Mac, then your default terminal running bash or zsh should be fine.

Geany

The recommended text editor for this class is Geany. You can download an installer from the releases page for Mac or Windows (make sure you get the arm64 installer if your Mac has an aarch64 CPU), and you can use your package manager to install it on Linux.

Java

We'll be using some new features that were previewed in Java 24 and will be finalized in Java 25. You should install Java 25 when it becomes available on September 16th, but until then you can use Java 24 and enable preview features when running your programs. We'll cover enabling preview features when we write our first program.

The Eclipse Foundation provides free builds of the JDK (Java Development Kit), which you can download from their releases page. Click "All Versions", then select your operating system, architecture, and version 25 (or 24 if 25 is not yet available). Choose the .MSI (Windows) or .PKG (Mac) installers. The .ZIP and .TAR.GZ archives require manual installation.

On Linux, you should check your package manager to see whether it has version 24 or 25 of Java. If not, then you can manually install the .TAR.GZ download. Please see me during office hours if you need help with this.

After you install java, you can confirm that it was installed correctly by restarting your terminal and running the command java -version. This should show the version number of your Java installation (24 or 25).

Uninstall Java

If you accidentally install the wrong version or need to upgrade to another version, then you may have to remove an existing installation. On Windows, you can follow these directions from Microsoft. On Mac, you will need to delete the installation from your /Library/Java/JavaVirtualMachines folder using the directions below. Be careful when accessing the /Library folder! Your OS normally hides this folder from you because deleting or modifying the wrong files can cause problems with your computer. Only delete the folder for the Temurin installation you want to get rid of.

  1. Open the Go to Folder dialog in Finder.
  2. Copy /Library/Java/JavaVirtualMachines into the search bar and open that folder. Make sure you copy this path exactly as shown, including the leading /. There are other folders named Library on a Mac, so taking a different shortcut may lead you to one that doesn't contain the Java installation.
  3. You should see a folder for each version of the Temurin JDK you've installed. Each folder's name should be in the format temurin-<version>.<jdk or jre>. For example, the Java 24 JDK might be named temurin-24.0.2+12.jdk. Delete the unwanted installations. You may need to enter your password to delete the folder.
